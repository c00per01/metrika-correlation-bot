# 🤖 Telegram-бот корреляций Яндекс.Метрики

Бот анализирует корреляции между целями Яндекс.Метрики. Показывает только умеренные и выше (|r| ≥ 0.4).

## Возможности

- ✅ Корреляции целей с выбранными главными (несколько = единая цель)
- ✅ Автоопределение периода (от первого срабатывания до сегодня)
- ✅ Ручная установка периода
- ✅ Среднее количество срабатываний в неделю для каждой цели
- ✅ Сохранение настроек (не теряются при перезапуске)
- ✅ Работает 24/7 на GitHub Actions бесплатно

## 📋 Как запустить (GitHub Actions — 24/7 бесплатно)

### 1. Создать GitHub репозиторий

1. Зайдите на [github.com](https://github.com) → **New repository**
2. Название: `metrika-correlation-bot` (или любое другое)
3. ⚠️ **Обязательно Private** — чтобы никто не видел workflow
4. Нажмите **Create repository**

### 2. Добавить секреты

В настройках репозитория: **Settings → Secrets and variables → Actions → New repository secret**

Добавьте 3 секрета:

| Имя секрета | Значение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен вашего бота из @BotFather |
| `METRIKA_OAUTH_TOKEN` | OAuth токен Яндекс.Метрики |
| `ALLOWED_USER_ID` | Ваш Telegram user ID |

### 3. Загрузить код

```bash
cd "D:\Бот телеграм Метрика корреляции"
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/metrika-correlation-bot.git
git push -u origin main
```

### 4. Проверить

1. Зайдите в **Actions** вашего репозитория
2. Workflow «Telegram Bot 24/7» должен запуститься автоматически
3. Бот заработает! Можете писать ему в Telegram.

## 🕹️ Команды бота

| Команда | Описание |
|---|---|
| `/start` | Приветствие и текущие настройки |
| `/counter <id>` | Установить ID счётчика |
| `/goals` | Показать все цели счётчика |
| `/main <id,id,...>` | Задать главные цели (через запятую) |
| `/period <from> <to>` | Задать период вручную (YYYY-MM-DD) |
| `/autoperiod` | Сбросить период на автоопределение |
| `/run` | Запустить анализ |
| `/status` | Текущие настройки |
| `/reset` | Сбросить все настройки |

## Пример использования

```
/counter 44147844
/goals
/main 255302693,257143059
/run
```

Бот автоматически определит период с первого срабатывания любой из главных целей до сегодня.

## Как это работает

1. Несколько выбранных главных целей объединяются **в одну** (сумма срабатываний за день)
2. Для каждой другой цели считается корреляция Пирсона с главной
3. Показываются только корреляции |r| ≥ 0.4 (от умеренной и выше)
4. Период определяется автоматически: от самого раннего первого срабатывания любой из выбранных главных целей до сегодня
5. Дополнительно: среднее кол-во срабатываний в неделю для каждой цели
